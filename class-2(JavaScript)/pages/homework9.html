[#9] 약간 복잡한 Array & Object 데이터 바인딩 2022. 07. 26

오늘은 모달창의 dimmed 영역을 누르면 모달창 닫는 기능과 client-side rendering 개발방식으로 템플릿으로 기본 정보만 입력되어있는 곳에 javaScript로 변수에 저장된 정보를 입력해주는 것을 배웠다.

dimmed 영역을 눌렀을 때 모달창이 닫히는 기능은 dimmed 영역을 눌렀을 때 닫히기는 하는데, 모달창을 눌렀을 때도 닫히게 되는데 이유는 이벤트 버블링이라는 현상 때문이었다. 어떤 html 태그에 이벤트 발생시 그 모든 상위요소까지 이벤트가 실행이 되는데 dimmed 안에 모달창 태그가 포함되어있기에 모달창을 눌렀을 때도 모달창의 상위요소인 dimmed도 누른걸로 인식된다고 한다. 그렇기에 해결책은 특정함수를 넣어주는 건데, 이번의 경우 e.taget 이라는 실제 클릭한 요소를 알 수 있는 함수를 활용해서 실제 dimmed 영역만 눌렀을 때 모달창 닫는 함수를 실행하도록 하였다. 관련 이벤트 함수 메모: e.taget(실제 클릭한 요소), e.currentTarget(지금 이벤트 리스너가 달린 곳을 알려줌), e.preventDefault() (실행하면 이벤트 기본 동작을 막아줌 - submit 막기), e.stopPropagation() (실행하면 내 상위요소로의 이벤트 버블링을 중단)

그리고 오늘은 카드들에 변수에 저장된 제목과 가격을 자바스크립트로 넣어주는 숙제를 했다. array 자료형과 object 자료형을 배웠고, 그 자료형 안에 들어가있는 자료를 선택하여 특정 html에 넣어주는 방법을 배웠다. 배운 것을 활용해서 주어진 object 자료형에 입력된 정보를 만들어둔 카드들에 넣어주는 기능을 만들었고, 반복되는 것들은 어제 배웠던 for 반복문을 활용해서 코드를 줄여보았다.

기능 개발을 할 때 항상 console.log를 활용해서 간단한 것부터 잘 나오는지 테스트를 해보면서 개발하는 것이 훨씬 쉽고 오류도 덜 나는 것 같다. 그동안 어렵고 이해가 쉽지 않아서 좀 덜 재밌었는데, 오늘은 그동안 배운 것들을 잘 활용한 것 같고, 이해도 잘 되어서 너무 재밌었다..! 중간중간 작년에 독학했을 때가 문득 생각나면서 아 그걸 이렇게 해결했으면 쉬웠겠다.. 라고 생각도 드는게 되게 신기하고 재밌었다. ㅎㅎ
